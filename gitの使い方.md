# Gitの使い方（毎回の作業手順）

このファイルでは、ファイルを変更したあとにGitHubにアップロードする手順を説明します。

---

## 基本の流れ（毎回やること）

```
変更する
    ↓
① 変更を保存する（git add）
    ↓
② 変更を記録する（git commit）
    ↓
③ GitHubにアップロードする（git push）
```

---

## 1. ターミナルを開く

VS Codeで `Ctrl + @` を押すと、画面の下にターミナル（黒い画面）が開きます。

---

## 2. 変更したいフォルダに移動する

```bash
cd /Users/shiozawatakumi/Documents/07_TENTO/TENTO_AI_TEST
```

このコマンドで `TENTO_AI_TEST` フォルダに移動します。

---

## 3. 変更を保存する（git add）

ファイルを編集したら、まずはGitに「このファイルを保存したい」と伝えます。

### 特定のファイルだけ保存する場合
```bash
git add index.html
```
（例：index.html だけを保存したいとき）

### すべての変更をまとめて保存する場合
```bash
git add .
```
（「.」は「今いるフォルダの全部」という意味です）

---

## 4. 変更を記録する（git commit）

保存した変更に「何を変えたか」というメモを付けて記録します。

```bash
git commit -m "変更内容をここに書く"
```

### メッセージの書き方の例

| 変更の種類 | メッセージの例 |
|-----------|---------------|
| 新しい機能を追加 | `"ポートフォリオにスキル追加"` |
| デザインを変更 | `"ヘッダーの色を青に変更"` |
| 文章を修正 | `"プロフィールの文章を修正"` |
| ファイルを追加 | `"README.mdを追加"` |

---

## 5. GitHubにアップロードする（git push）

記録した変更をGitHubに送信します。

```bash
git push
```

初めてのときだけ、代わりに以下のコマンドを使います（一度設定すれば次からは `git push` だけで大丈夫）。

```bash
git push -u origin main
```

---

## 6. よくある質問

### Q. 今の状態を確認したい
```bash
git status
```

今どんなファイルが変更されているか、何を保存しようとしているかが表示されます。

### Q. これまでの記録を見たい
```bash
git log --oneline
```

今までに記録した変更の一覧が表示されます。

### Q. 保存する前に間違いに気づいた
```bash
git diff
```

変更した内容が表示されるので、確認してから保存できます。

## 7. 今日の作業の流れ（例）\n
例えば、ポートフォリオサイトのデザインを変更した場合の流れです。

```bash
# 1. フォルダに移動
cd /Users/shiozawatakumi/Documents/07_TENTO/TENTO_AI_TEST

# 2. 今の状態を確認（変更したファイルがあるかチェック）
git status

# 3. ファイルを編集（VS Codeで編集する）
# → index.html を開いて、色やレイアウトを変更する

# 4. 変更を保存（git add）
git add index.html

# 5. 変更を記録（git commit）
git commit -m "ポートフォリオの背景色を変更"

# 6. GitHubにアップロード（git push）
git push
```

### 別の例：新しいファイルを追加した場合

```bash
cd /Users/shiozawatakumi/Documents/07_TENTO/TENTO_AI_TEST

# 新しいファイルを作ったら、まずはgit add . で全部保存
git add .

# コミット
git commit -m "作品紹介ページを追加"

# プッシュ
git push
```

### よく使う3つのコマンドまとめ

| 順番 | コマンド | 意味 |
|------|---------|------|
| 1 | `git add .` | ファイルの変更を保存する準備 |
| 2 | `git commit -m "説明"` | 変更を記録する |
| 3 | `git push` | GitHubにアップロードする |

この3つを覚えておけば毎日の作業は大丈夫！
