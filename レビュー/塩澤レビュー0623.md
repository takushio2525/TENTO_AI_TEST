## 総括

`TENTO_AI_TEST` 配下で、AI コーディングツールに **ポートフォリオサイト作成** と **オセロゲーム作成**、および周辺の **Git/README ドキュメント整備** を依頼し、その挙動を検証した記録。

| 8 | モダン UI のオセロゲーム（CPU 4 段階）作成 | `02test_othello/index.html` |

### 主な問題点

- **出力パス指定が安定しない（最重要）**: 絶対パスを指定しても、ワークスペースとして開いていた `07_TENTO/` 直下にファイルが作られるなど、指定ディレクトリと食い違うケースが複数回発生。最後のオセロでも結局パス指定通りにならなかった。
- **ディレクトリ名のスペース／`_` の扱い**: rename を含む diff が適用できず、`workspace 外の path` エラーも発生。AI は別アプローチで自己回復できていた。
- **「説明します」と言って説明しない**: GitHub 公開手順について「手順を説明します」と書いたのに、実際の説明本文がどこにもなかった。
- **軽微な記述漏れ**: 手順解説 md に章はあるが内容が空、などの抜けがあった。

### 良かった点

- ツール呼び出しが失敗しても、原因を推測して**別アプローチで自力回復**できていた。
- Tailwind 適用後の **UI のクオリティは高め**。
- 公開手順の追記や空章の補完など、**指摘への追従**は概ね良好。
